# D1 — Corruption Mechanism Specification

**Project:** When Does Data Quality Predict Model Failure? A Benchmark for Knowing Before It Breaks
**Version:** 1.0 (Frozen; §15 item 3 updated 2026-07-17 per binary-label spot-check resolution)

---

# 1. Objective

Phase 1 generates realistic, reproducible, and measurable data-quality degradation scenarios simulating common failure modes observed in real-world machine learning systems. The corruption framework must:

1. Represent major data-quality dimensions.
2. Produce measurable degradation trajectories.
3. Support cross-dataset comparison.
4. Remain reproducible across experimental runs.

All corruption operations are applied after train/test splitting, per D2 (Split-Then-Corrupt Protocol).

---

# 2. Supported Data Quality Dimensions

The benchmark operationalizes five of the six data-quality dimensions defined in the project taxonomy.

| Dimension    | Mechanisms                  |
| ------------ | ---------------------------- |
| Completeness | MCAR, MAR, MNAR              |
| Accuracy     | Gaussian Noise, Label Noise  |
| Consistency  | Categorical Swap             |
| Validity     | Validity Violations          |
| Uniqueness   | Duplicate Records            |
| Timeliness   | Out of scope (§13)           |

# 3. Notation

$$D=(X,Y), \quad X \in \mathbb{R}^{n \times p}, \quad Y \in \{0,1\}^{n}$$

$$R \in \{0,1\}^{n \times p}, \quad R_{ij}=1 \text{ indicates } X_{ij} \text{ observed}$$

$$\alpha \in [0,\alpha_{max}] \text{ — corruption severity, calibrated to Phase 0 empirical defect rates (see §10)}$$

**Model families (fixed for Phase 1–3):** XGBoost, Random Forest, Logistic Regression, MLP. Four families, chosen for standard coverage (tree ensemble, bagged tree, linear, neural) at a count that keeps per-family sample sizes usable after LODO folding.

**Corpus assumption — RESOLVED:** all 17 datasets are treated as binary post-Phase-0-binarization. This assumption was verified via a binary-label spot-check completed 2026-07-17 (`preprocess_to_processed.py`, see §15 item 3). All 17/17 datasets confirmed binary post-processing; §6.2's general multi-class fallback is not invoked for any dataset in the current corpus.

$\alpha$ never approaches 1 for any mechanism: a fully-missing feature, fully-flipped label set, or fully-duplicated dataset is degenerate, not "high severity," and produces no usable dose-response signal.

# 4. Missingness Mechanisms

Rubin's (1976) taxonomy.

## 4.1 MCAR

$$P(R_{ij}=0 \mid X,Y)=\pi_j(\alpha)$$
Bernoulli masking applied independently to each cell.
**Validation:** target missingness rate achieved; no significant association between missingness indicators and observed features.

## 4.2 MAR

$$P(R_{ij}=0)=\sigma(\alpha z(X_{ik})), \quad k \neq j$$
$k$: fixed conditioning feature, highest absolute correlation to $j$, selected once per dataset.
**Metadata:** `conditioning_feature`, `conditioning_strength`.
**Validation:** missingness significantly associated with conditioning feature; target rate achieved.

## 4.3 MNAR

$$P(R_{ij}=0)=\sigma(\alpha z(X_{ij}))$$
**Validation:** missingness significantly associated with original (pre-corruption) feature value; target rate achieved. **This check is blocking, not advisory** — MAR/MNAR are only meaningfully distinct from MCAR if the dependence structure survives into corrupted data.

# 5. Feature Noise

## 5.1 Additive Gaussian Noise

$$X'_{ij}=X_{ij}+\epsilon_{ij}, \quad \epsilon_{ij}\sim N(0,(\alpha s_j)^2), \quad s_j = \text{train-fold SD of feature } j$$
Applied only to continuous features. **Validation:** observed noise variance matches target.

## 5.2 Categorical Swap

A categorical value is replaced by another category sampled from the empirical marginal, excluding the original category, for a proportion $\alpha$ of values.
**Validation:** achieved corruption rate matches target; marginal distribution approximately preserved.

## 5.3 Quantization / Bin Collapse

$$k = k_{max} - \alpha(k_{max}-k_{min}), \quad k_{min}\ge 3$$
Equal-frequency binning on train fold. **Validation:** achieved bin count matches target.

---

# 6. Label Noise

## 6.1 Uniform Label Noise

$$P(Y'_i \neq Y_i)=\alpha$$
**Validation:** observed flip rate equals target.

## 6.2 Class-Conditional Label Noise

$$P(Y'=y'\mid Y=y)=T_{yy'}$$
For binary datasets (the corpus-wide case, confirmed §3): $\alpha_{0\rightarrow1}$, $\alpha_{1\rightarrow0}$ define asymmetric flip rates. **Calibration:** the asymmetry ratio is estimated from a Cleanlab confident-learning pass per dataset, rather than asserted from domain intuition — keeps this mechanism consistent with this document's anchoring discipline. Requires no probe model in the binary case.

**Fallback (multi-class):** not invoked for the current 17-dataset corpus per §3's resolved spot-check. Retained here for completeness / future corpus extensions: general $T$ requires a confusable-pair rule derived from a frozen, one-time diagnostic model *not* among the four families in §3, run once on clean data — avoiding the same probe-circularity risk that motivates deferring §6.3.

**Validation:** observed transition matrix (or $\alpha_{0\to1}, \alpha_{1\to0}$) matches target.

## 6.3 Boundary-Aware Label Noise — Deferred

Noise concentrated near the decision boundary is deliberately not implemented in Phase 1. Locating the boundary requires a probe model, which risks circularity if that probe overlaps one of the four evaluated families (§3). This is a scoping decision, not an oversight: ship uniform and class-conditional noise first to establish a clean reference result, then reconsider boundary-aware noise as a controlled follow-up once that reference exists.

# 7. Distribution Shift

Moreno-Torres et al. (2012); Quiñonero-Candela et al. (2009).

## 7.1 Covariate Shift

$$P'(X)\neq P(X), \quad P'(Y|X)=P(Y|X)$$
$$g(x)=PC_1(x) \text{ (train-fold standardized numeric features)}, \quad P(select)\propto \exp(\alpha g(x))$$
PC1 is the frozen, sole generator — deterministic and reproducible. Note that PC1 is the direction of maximum variance, not necessarily the direction most predictive of $Y$: a shift along it can produce large PSI with little actual degradation, or the reverse. This is retained deliberately rather than replaced with a supervised (e.g. PLS1) alternative, since a benchmark where drift magnitude and degradation magnitude always moved together would be trivially solvable in a way this project is specifically designed to avoid.
**Validation:** PSI and Wasserstein distance increase with severity; $P(Y|X)$ unchanged (spot-check on retained rows).

## 7.2 Label Shift

$$P'(Y)\neq P(Y), \quad P'(X|Y)=P(X|Y)$$
$$\rho'(\alpha) = (1-\alpha)\rho + \alpha\rho_{target}$$
$\rho$: natural class prior. $\rho_{target}$: maximum permitted shifted prior.

This document freezes the functional form and a binding constraint on $\rho_{target}$, not its numeric value. $\rho_{target}$ must not push a fold's minority count below what Phase 0's downstream requirements (fold arithmetic, group-aware conformal cell sizes) need at realistic post-split sample sizes — a distinct, per-fold constraint from the corpus-admission 10% floor, not a restatement of it. D4 derives the actual per-dataset $\rho_{target}$ from this constraint; it must not be set independently.
**Validation:** target class prior achieved; $P(X|Y)$ preserved.

## 7.3 Slice-Conditional Concept Shift

$$P'(Y|X\in S) \neq P(Y|X\in S), \quad S \text{ fixed, } X \text{ unchanged}$$
Labels reassigned only within $S$; chosen once, frozen, documented per dataset. Approved as the operationalization of concept shift for Phase 1 — H1 explicitly names concept shift as a regime where quality/provenance features should help, and dropping it entirely would silently narrow H1.
**Severity anchor:** tied to D5 — measured real label-conditional shift magnitude in the natural-shift anchor datasets (road-safety, diabetes-readmission).
**Validation:** target conditional label change achieved within $S$; KS test confirms no row resampling occurred during slice selection (note: since only $Y$ is touched, $P(X)$ invariance holds by construction — this check validates slice selection, not "concept shift stayed contained" more broadly).

# 8. Validity Violations

Addresses the Validity dimension, previously uncovered by any mechanism.

$$X_{ij}\rightarrow v_{invalid} \text{ for proportion } \alpha \text{ of selected cells}$$
$v_{invalid}$ chosen per-feature from a one-time domain constraint annotation: known sign/range constraint (age, counts, amounts) → negation or large offset; no documented constraint → value beyond train-fold IQR-based bound.
**Severity anchor:** naturally occurring validity violations already documented in Phase 0 (e.g. HELOC's raw sentinel values) anchor $\alpha_{max}$ to a measured rate rather than an asserted one.
**Ownership:** Kishore owns the one-time domain-constraint annotation pass across all 17 datasets. Estimated effort: ~half a day. Scheduled before D3 implements this mechanism.
**Validation:** target corruption rate achieved; injected values violate the predefined validity rule by construction.

---

# 9. Uniqueness Violations

Addresses the Uniqueness dimension, also previously uncovered. Duplicating a row is equivalent, in an empirical-risk-minimization sense, to increasing its weight in the training objective — this is the causal mechanism by which uniqueness violations degrade performance, distinct from the noise/removal mechanisms elsewhere in this document.

## 9.1 Global Duplication

A proportion $\alpha$ of records sampled uniformly with replacement, appended as exact duplicates (no jitter — would conflate with feature noise). Deliberately retained as a weak/near-null-effect regime: a defect present but producing $\Delta M \approx 0$ in most datasets is a hard-negative case, not signal to engineer away, and keeps the benchmark from being trivially solvable.
**Validation:** observed duplicate rate matches target; duplicates approximately uniform across feature space.

## 9.2 Slice-Conditional Duplication

A proportion $\alpha$ of records duplicated only within $S$ — the same slice already frozen for concept shift (§7.3), reused rather than independently selected, so D11's stratified evaluation can ask whether a given region degrades under concept shift and duplication, or only one.
**Validation:** duplicate rate within $S$ matches target; duplicates remain concentrated within $S$.

**Severity anchor (both variants):** naturally occurring duplicate rows already documented in Phase 0 (e.g. HELOC's 588 duplicate rows) anchor $\alpha_{max}$.

---

# 10. Severity Calibration

Severity levels are not arbitrary percentages. Each mechanism is calibrated to empirically observed defect rates measured during Phase 0:

- Missingness → natural per-column missingness rates (e.g. road-safety's ~8.96%)
- Additive noise / uniqueness → HELOC's naturally occurring duplicate rows, used both as a noise-magnitude estimate (§5.1) and a duplication-rate anchor (§9)
- Validity → HELOC's naturally occurring sentinel values
- Label noise → Cleanlab-estimated per-dataset label error rate

This document defines calibration methodology only. D4 defines final numeric severity values per dataset, checked against the actual Phase 0 catalog output rather than asserted here.

---

# 11. Validation Requirements

Every corruption mechanism must satisfy two independent requirements:

**11.1 Corruption Fidelity** — observed corruption magnitude matches requested severity (missingness rate, flip rate, duplicate rate, shift magnitude).

**11.2 Mechanism Fidelity** — observed corruption structure matches the theoretical mechanism definition (MAR depends on conditioning feature; MNAR depends on its own value; label shift preserves $P(X|Y)$; covariate shift preserves $P(Y|X)$; concept/uniqueness slice effects are concentrated in $S$).

A corruption run failing either requirement is invalid and must be flagged by D8, not silently accepted.

---

# 12. Reproducibility Requirements

- Fixed random seeds.
- Fold-local parameter estimation — all corruption parameters ($s_j$, conditioning feature statistics, bin edges, PC1 direction, domain-constraint thresholds, $\rho$) estimated on train fold only, applied to both folds.
- Deterministic severity definitions.
- Complete corruption metadata logging (conditioning features, achieved rates, frozen constants) into the D7 manifest.
- Full regeneration through the D9 pipeline.

---

# 13. Out-of-Scope Mechanisms

**Cross-Field Consistency Violations** (e.g. country–postal code mismatches, employment–salary contradictions) are excluded: they require domain-specific, per-dataset-pair constraints that don't generalize across 17 heterogeneous domains and would risk becoming a second, less-principled curation effort layered on Phase 0's.

**Timeliness Degradation** is excluded: most datasets lack reliable timestamps for realistic temporal-degradation simulation. Distribution shift (§7) captures the consequences of temporal drift without requiring a temporal axis; genuine temporal staleness is noted as an external-validity limitation.

---

# 14. Experiment Budget

14 sub-mechanisms total: 3 missingness + 3 feature noise + 2 label noise (boundary-aware deferred, not counted) + 3 distribution shift + 1 validity + 2 uniqueness.

$$17 \text{ datasets} \times 14 \text{ mechanisms} \times 10 \text{ severities} \times 4 \text{ families} = 9{,}520 \text{ base rows}, \quad \times\ 5 \text{ seeds} = 47{,}600 \text{ total rows}$$

Illustrative at this level of granularity — D4 owns final grid sizing and may adjust seed count or severity resolution against actual compute budget.
