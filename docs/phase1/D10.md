# D10 — Quality and Drift Metric Taxonomy

---

# 1. Purpose

## 1.1 Objective

D10 defines the complete **quality and drift measurement taxonomy** used throughout the project.

Every quantity specified in this document becomes an input feature to the Phase 2 meta-model, whose objective is to predict model performance degradation after data corruption.

Unlike D3, which generates corrupted datasets, D10 specifies **how corruption is measured**. Consequently, the scientific contribution of the meta-model depends directly upon the quality of this measurement framework — and that dependency is exactly why every metric below carries its symbolic definition, not just a description: §5 (Decision 5) requires implementations to conform to the mathematical definitions specified here, so those definitions have to actually be present.

## 1.2 Scope

D10 specifies: quality metrics, drift metrics, information metrics, aggregation strategy, estimator choices, feature applicability, reproducibility requirements, and implementation-independent mathematical definitions.

It does **not** specify: provenance-weighted features, feature engineering, preprocessing, meta-model architecture, or explainability analyses. Those belong to later phases (§11).

## 1.3 Design Goals

**Goal 1 — Completeness.** Capture every important manifestation of dataset corruption _that clears the complementarity bar in Goal 2_ — completeness is bounded by Goal 2, not a license to keep adding plausible-sounding metrics. The taxonomy stops at twelve definitions on purpose.

**Goal 2 — Complementarity.** Each metric must contribute unique information rather than duplicate existing measurements.

**Goal 3 — Cross-Dataset Consistency.** The same measurement framework must operate correctly on datasets ranging from roughly five to more than three hundred features, producing a fixed-length representation regardless.

**Goal 4 — Reproducibility.** Independent implementations should produce identical metric values from identical corrupted datasets — which requires the mathematical definitions in §4 to be complete enough to implement from without guessing.

---

# 2. Design Philosophy

The objective of D10 is **not** to maximize the number of quality metrics. Instead, the objective is to maximize the amount of independent information available to the Phase 2 meta-model while maintaining computational efficiency. The taxonomy therefore emphasizes **complementarity rather than redundancy**.

The five categories jointly capture: completeness degradation, distributional drift, statistical significance, information loss, and label-space degradation. Each category exists because it measures something that at least one other category cannot — Distribution metrics detect marginal movement; Information metrics detect deterioration in feature-label relationships that marginal movement can miss entirely; Missingness metrics distinguish MCAR from MAR and MNAR, which a single missing-rate scalar cannot; Statistical tests distinguish real drift from sampling noise, which magnitude-only Distribution metrics cannot; Label metrics verify that corruption has not unintentionally altered class-space assumptions, independent of what the meta-model does with the rest.

Because every metric must justify its existence against this bar, the taxonomy intentionally stops at twelve mathematical metric definitions rather than expanding into dozens of partially redundant measurements.

---

# 3. Governing Principles

## 3.1 Fold Locality

All metrics are computed under the fold-local protocol established in D2. Every metric is computed from exactly one corrupted realization and paired only with the corresponding performance measurement obtained from that same realization. Metrics and performance values originating from different corruption draws are never paired.

## 3.2 Comparative Metrics

Comparative metrics require two distributions:

- **Reference distribution** — clean training fold
- **Target distribution** — corrupted testing fold

This mirrors the intended deployment scenario: a model trained on historical clean data, deployed against degraded production data. The reference distribution is always the clean _train_ fold — never a pre-corruption version of the _same_ test fold, and never recomputed after corruption is applied.

## 3.3 Intrinsic Metrics

Intrinsic metrics require no reference distribution. They describe properties of the corrupted dataset itself. These include Missingness metrics and Class metrics.

## 3.4 Feature-Level Metrics

Most metrics are computed independently for every feature. Feature-level values are subsequently aggregated into fixed-length vectors (§5). This guarantees that datasets with different dimensionality generate identical meta-model input lengths.

# 4. Metric Taxonomy

The taxonomy consists of five complementary categories, twelve mathematical metric definitions in total.

## 4.1 Missingness

**Purpose.** Measures completeness degradation. Most directly associated with the MCAR / MAR / MNAR mechanisms defined in D1.

**Metric 1 — Missing Rate.** For feature _j_, over _n_ rows in the corrupted test fold:

`r_j = (1/n) · Σᵢ 1[x_ij is missing]`

Computed independently for every feature. Dataset representation: mean, max (§5).

**Metric 2 — Pattern Entropy.** Missingness often occurs in structured combinations rather than independently. Represent each row's missingness as a binary pattern vector across all _p_ features, and compute the Shannon entropy over the empirical distribution of unique patterns:

`H_pattern = − Σ_k P(pattern_k) · log₂ P(pattern_k)`

Low entropy indicates structured missingness (consistent with MNAR); entropy approaching `log₂(2^p)` indicates missingness closer to independent-per-feature (consistent with MCAR). Produces one dataset-level scalar — no aggregation needed.

**Metric 3 — Missingness Correlation Structure.** Pairwise association between the missingness indicators of feature pairs (j, j′), computed via the **φ-coefficient** (mathematically equivalent to Pearson correlation on binary indicator variables — Decision 11) and reported as the mean absolute off-diagonal value of the resulting p×p correlation matrix:

`ρ̄_miss = (2 / (p(p−1))) · Σ_{j<j′} |φ(m_j, m_j′)|`

Produces one scalar.

**Scientific motivation.** Missing rate alone cannot distinguish MCAR from MAR/MNAR. Pattern entropy and correlation structure explicitly capture missingness _mechanism_, not merely missingness magnitude.

---

## 4.2 Distribution

**Purpose.** Measures marginal feature drift. Every metric below is computed independently per feature, aggregated via mean/max (§5).

**Metric 4 — Population Stability Index (PSI).** For reference distribution _P_ (clean train fold) and target distribution _Q_ (corrupted test fold), binned into _k_ bins:

`PSI = Σ_k (Q_k − P_k) · ln(Q_k / P_k)`

Continuous variables: 10 equal-frequency (quantile) bins, computed only from the clean training fold (Decision 4). Categorical variables: native categories. **Zero-count-bin fallback (part of Decision 4):** any bin with `Q_k = 0` is assigned a floor probability of `1/(2n)` before the log term is evaluated, to prevent `PSI = ∞`. Without this floor, any sparsely-populated bin in the corrupted fold produces an undefined or infinite value — this is a required part of the metric's definition, not an optional implementation nicety.

**Metric 5 — Wasserstein Distance.** Continuous variables only. First-order (earth mover's) distance between the empirical reference and target distributions:

`W₁(P, Q) = ∫ |F_P(x) − F_Q(x)| dx`

computed via the standard order-statistics formula on the empirical CDFs. Advantages: no discretization required, respects feature scale.

**Metric 6 — Jensen–Shannon Divergence.** Applicable to all feature types (continuous, via the same bins as PSI; categorical, via native categories):

`JS(P‖Q) = ½·KL(P‖M) + ½·KL(Q‖M), where M = ½(P+Q)`

Advantages: symmetric, bounded (`[0, ln 2]`), directly comparable across datasets with different feature scales.

**Scientific motivation.** The three metrics complement one another:

| Metric        | Primary strength                   |
| ------------- | ---------------------------------- |
| PSI           | Industry-standard drift metric     |
| Wasserstein   | Scale-aware, binning-free distance |
| JS Divergence | Bounded, cross-dataset-comparable  |

**Excluded — Maximum Mean Discrepancy.** Excluded because kernel-bandwidth selection does not transfer cleanly across 17 heterogeneous datasets of very different dimensionality and scale, no single bandwidth rule (median heuristic or otherwise) is validated to behave consistently across the corpus, and MMD's O(n²) cost per fold per feature-set is not tractable at the 47,600-row experiment budget scale without a compute-budget renegotiation.

**Excluded — Energy Distance.** Correlates highly with Wasserstein in the univariate marginal setting used here and does not add distinguishable signal at this feature-level granularity. Deferred to a contingent Phase 2.5 ablation, not included in the frozen taxonomy now.

**Excluded — Drift Detection Method (DDM).** Designed for streaming/sequential online monitoring with running error-rate tracking; structurally incompatible with the static-batch corrupted-fold design D2 established.

---

## 4.3 Statistical Tests

**Purpose.** Measures statistical significance rather than drift magnitude. Computed per feature, aggregated via mean/max (§5).

**Metric 7 — Kolmogorov–Smirnov Statistic.** Continuous variables only:

`D = sup_x |F_P(x) − F_Q(x)|`

**Metric 8 — Chi-Square Statistic.** Categorical variables only (Decision 10 — continuous variables are excluded here entirely, not binned-and-included; KS covers the equivalent role for continuous features):

`χ² = Σ_i (O_i − E_i)² / E_i`

where `O_i` is the observed count in category _i_ of the corrupted test fold and `E_i` is the expected count under the reference (train-fold) category proportions, scaled to the test fold's size.

**Scientific motivation.** Distribution metrics (§4.2) answer "how much did the distribution move?" Statistical tests answer "was the observed movement statistically significant, or fold-size noise?" Both questions are necessary, particularly at the low-severity end of D4's grid, where a marginal PSI/Wasserstein movement may or may not be distinguishable from sampling noise — exactly the boundary where PAPE/CBPE-style estimators are known to struggle.

---

## 4.4 Information

**Purpose.** Measures degradation in predictive information. Computed independently per feature, aggregated via mean/max (§5).

**Metric 9 — Entropy Shift.**

`ΔH_j = H(X_j^corrupted) − H(X_j^reference)`

Continuous variables: discretized using the **same bins as PSI** (Decision 8 — equal-frequency, 10 bins, reference train-fold bins), not a second independent binning scheme. Categorical variables: native categories, same as PSI. Reusing PSI's bin edges keeps the two metrics internally consistent and reuses the same cached bin-edge artifact (§9).

**Metric 10 — Mutual Information Change.**

`ΔMI_j = MI(X_j^corrupted; Y) − MI(X_j^reference; Y)`

Estimator (Decision 9): `sklearn.feature_selection.mutual_info_classif`, with `discrete_features` set per actual feature type. Named explicitly because MI estimates are estimator-sensitive — two different implementations can produce materially different values from the same data.

**Scientific motivation.** A feature's marginal distribution may remain almost unchanged (low PSI/Wasserstein) while its predictive relationship to the label deteriorates substantially — exactly the covariate/concept-shift boundary case H1 is built around. Mutual information detects degradation invisible to conventional drift metrics, and directly addresses a known blind spot in drift-metric-only approaches (NannyML's own published negative result on drift-only performance estimation).

---

## 4.5 Class

**Purpose.** Measures label-space degradation. Produces intrinsic dataset-level scalars — no aggregation needed.

**Metric 11 — Imbalance Ratio.**

`IR = n_majority / n_minority`

**Metric 12 — Label Entropy.**

`H(Y) = − Σ_c P(y=c) · log₂ P(y=c)`

**Scientific motivation.** These function primarily as **manipulation checks**, not primary predictive features. H1's regime-conditional framing requires "label-marginal change near zero at realistic-but-high severity" — IR and label entropy, computed pre/post corruption, are how that condition gets verified empirically per corruption draw, not assumed. If a draw nominally targeting covariate shift moves label entropy substantially, that's evidence the corruption leaked into label-marginal territory.

# 5. Feature Aggregation

## 5.1 Motivation

Eight of the twelve metric definitions are computed **per feature**, not once per dataset. Without a formal aggregation policy, a 5-feature dataset and a 300-feature dataset would produce feature vectors of different lengths — unusable by a single pooled meta-model.

## 5.2 Feature-Level Metrics

Missing Rate, PSI, Wasserstein Distance, Jensen–Shannon Divergence, KS Statistic, Chi-Square Statistic, Entropy Shift, Mutual Information Change.

## 5.3 Aggregation Strategy

Each feature-level metric is summarized using two aggregate statistics, both retained as independent input features:

- **Mean** — represents diffuse degradation affecting many features simultaneously (broad covariate shift, MCAR missingness, mild global corruption).
- **Maximum** — represents concentrated degradation affecting a small number of features (validity violations, localized feature corruption, severe MNAR).

Collapsing to mean alone would blur exactly the kind of mechanism-signature distinction that motivated including pattern entropy alongside rate in §4.1 — the same reasoning applies here.

## 5.4 Why Standard Deviation Was Excluded

Evaluated and excluded: it provides weaker independent information than mean and max, is highly correlated with the existing aggregates, and increases dimensionality without a clear scientific justification beyond what mean/max jointly capture.

## 5.5 Dataset-Level Metrics

Missingness Pattern Entropy, Missingness Correlation Structure, Imbalance Ratio, and Label Entropy are already computed once per dataset and require no aggregation.

# 8. Decision Log

**Decision 1 — Taxonomy freeze.** Frozen at five categories, twelve mathematical metric definitions. Future additions require the same documented-exception protocol used for Phase 0's corpus decisions (dated amendment, stated rationale, no silent insertion).

**Decision 2 — Excluded metrics.** MMD, Energy Distance, and DDM excluded from v1.0 (see §4.2 for full per-metric reasoning, including the tie to the 47,600-row compute budget for MMD specifically). May be reconsidered only through a future, documented ablation study — not added opportunistically during D3 implementation.

**Decision 3 — Reference distribution.** Comparative metrics always compare the clean training fold (reference) against the corrupted testing fold (target). The reference is never recomputed after corruption, and is never a pre-corruption version of the _same_ fold being evaluated — it is always the separate, clean train fold (D2 Decision 2).

**Decision 4 — PSI binning, including the zero-count-bin fallback.** Continuous variables: quantile bins, ten bins, edges computed only from the clean training fold and reused unchanged across every corruption realization for that fold — never re-binned per draw. Categorical variables: native categories. **Fallback:** any bin with `Q_k = 0` receives a floor probability of `1/(2n)` before the log term is evaluated, applied consistently and logged per computation, to prevent `PSI = ∞`. This fallback is part of the frozen definition, not an implementation detail left to D3's discretion.

**Decision 5 — Mathematical authority.** Implementations must conform to the mathematical definitions specified in §4 of this document. Software libraries are implementation choices (§9) and are not part of the design specification itself.

**Decision 6 — Provenance-weighted features (Option A, resolved 2026-07-18).** Provenance-weighted features remain outside D10, deferred to Phase 2 feature engineering (or a future D11 addendum) rather than specified as a sixth D10 category. This was flagged as a decision needing David's input specifically, since it determines where the project's most distinctive novelty claim gets formally specified — Kishore's Option-A decision is the working default going into Phase 2 planning, still pending David's explicit confirmation (§12).

**Decision 7 — Feature aggregation.** All eight feature-level metrics aggregated via mean and max, both retained as separate columns; standard deviation excluded (§5.4).

**Decision 8 — Entropy discretization.** Entropy Shift uses identical discretization to PSI (same cached bin edges), guaranteeing internal consistency across the two information-theoretic/distributional metrics that share a binning step.

**Decision 9 — Mutual information estimator.** `sklearn.feature_selection.mutual_info_classif`, with `discrete_features` set per actual feature type. Named explicitly and version-pinned (§9) because MI estimates are estimator-sensitive.

**Decision 10 — Chi-square applicability.** Categorical variables only. Continuous variables are evaluated exclusively via the Kolmogorov–Smirnov statistic instead — the two statistics partition the feature set by type rather than overlapping.

**Decision 11 — Missingness correlation.** φ-coefficient, explicitly named (mathematically equivalent to Pearson correlation on binary indicator variables) to prevent an implementer substituting Spearman or another rank correlation without documented reason.

**Decision 12 — Missing feature types.** Unavailable metrics (e.g., chi-square's mean/max on a dataset with zero categorical features) are recorded as **NaN** — never replaced with zero (which would misleadingly read as "no drift detected"), never dropped from the schema (which would break the pooled meta-model's fixed-schema requirement), and never silently ignored. Imputation happens during Phase 2 preprocessing, not here. **Forward note for Phase 2 (not resolved in D10):** consider a per-aggregate `was_measurable` indicator flag alongside the imputed value, so the meta-model and SHAP attribution can distinguish "structurally not applicable" from "measured as unusually low."

# 9. Reproducibility Requirements

- Reference distributions remain fixed within each fold-split; bin edges (Decision 4) are computed once per dataset per split, cached alongside the corruption-seed hash from D2's caching-risk mitigation, so a PSI or entropy-shift value is always traceable to the exact bin edges and seed that produced it.
- PSI and Entropy Shift reuse identical cached bin edges (Decision 8) — never two independently-computed binning schemes.
- Every corruption realization records its random seed.
- All metric computation runs through a single pipeline entry point, `(reference_fold, corrupted_fold) → metric_vector`, exercised identically regardless of which mechanism/severity/seed produced the corrupted fold.
- Aggregation (§5) occurs only after feature-level metric computation — never interleaved with it.
- Estimator and library versions are recorded in code (notably `scikit-learn`'s version, since `mutual_info_classif`'s internal estimator has changed across releases).
- _Non-binding implementation note:_ Evidently's drift preset already computes PSI, KS, and JS divergence natively and may be reused where convenient — a D3 implementation convenience, not a D10 design commitment (consistent with Decision 5). Missingness pattern entropy, correlation structure, entropy shift, and ΔMI are not covered by Evidently's defaults regardless.

These requirements exist so that independent implementations generate identical metric vectors from identical corrupted datasets — the practical test of Decision 5.

# 10. Scientific Contributions

D10 provides a unified measurement framework for tabular data corruption. Rather than relying solely on conventional drift metrics, it combines completeness degradation, marginal distribution shift, statistical significance, information-theoretic degradation, and label-space verification — each category capturing a distinct failure mode the others cannot fully explain. The resulting representation is complementary rather than redundant, while remaining computationally tractable across a heterogeneous, 17-dataset, 5-to-300-feature corpus. The taxonomy supports cross-dataset generalization while deliberately avoiding metric proliferation — twelve justified definitions, not twenty or fifty plausible-sounding ones.
