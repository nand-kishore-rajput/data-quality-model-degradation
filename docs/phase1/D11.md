# D11 v1.2 — Prediction Target and Evaluation Specification

## Purpose

D11 defines **what the Phase 2 meta-model predicts** and **how its performance will be evaluated**.

While D10 defines the **input features** (quality and drift metrics), D11 defines the **prediction target**, the **evaluation protocol**, the **baseline comparisons**, and the **statistical tests** required to validate the project's research hypotheses.

Together, D10 and D11 completely define the learning problem:

- **D10:** What information is available to the meta-model.
- **D11:** What the meta-model predicts and how success is measured.

---

# Prediction Target

## Primary Target

The meta-model predicts **Relative ΔM**, representing the relative performance degradation caused by data corruption.

```text
Relative ΔM = (Mclean − Mcorrupted) / Mclean
```

where:

- **Mclean** = performance on the clean test fold
- **Mcorrupted** = performance on the corrupted test fold

A positive value always indicates performance degradation.

Example:

```text
Clean Accuracy      = 0.90
Corrupted Accuracy = 0.81

Relative ΔM

= (0.90 − 0.81) / 0.90

= 0.10

= 10% degradation
```

---

## Why Relative ΔM?

Different datasets use different evaluation metrics.

Examples:

- AUC
- AUPRC

Raw performance drops cannot be compared fairly across these metrics.

Using Relative ΔM expresses degradation as a **fraction of each dataset's own baseline performance**, making the prediction target more comparable across heterogeneous datasets.

Raw ΔM is still recorded for analysis, but **Relative ΔM is the only target used to train the meta-model.**

---

## Secondary Target

The continuous Relative ΔM prediction is converted into degradation detection using predefined thresholds.

Thresholds:

- 1%
- 3%
- 5%
- 10%

These represent increasing levels of degradation:

- Negligible
- Mild
- Moderate
- Severe

The binary detection task is **derived from the regression output**.

No separate classifier is trained.

---

# Training Sample Generation

Each corruption experiment produces **one training sample**.

Example:

```text
Dataset      : Adult
Mechanism    : Missingness
Severity     : 5
Seed         : 3
Model Family : Random Forest
```

For this experiment:

Compute

- 12 D10 quality metrics
- Relative ΔM

Store

```text
12 Quality Metrics
+
Relative ΔM
+
Experiment Metadata
```

This becomes one row in the Phase 2 manifest.

Repeating the process across all experiments produces approximately **47,600 training samples**.

---

# Clean Baseline

For every dataset and model family:

1. Train the base model once using the clean training fold.
2. Evaluate once using the clean test fold.
3. Store this as **Mclean**.

This clean baseline is:

- computed once,
- deterministic,
- reused for every corruption experiment.

It is **not** recomputed for every corruption seed.

---

# Corruption Seeds

Corruption seeds introduce randomness only in the corruption process.

They do **not** change:

- train/test split,
- model training,
- clean baseline.

Each seed produces a unique corrupted dataset.

Each experiment is stored independently.

Example:

```text
Dataset A
Severity 5

Seed 1 → Row 1

Seed 2 → Row 2

Seed 3 → Row 3

...

Seed 5 → Row 5
```

No averaging occurs before storage.

---

# Meta-model

A single pooled meta-model is trained using data from all datasets.

Additional metadata supplied to the model includes:

- model family
- primary evaluation metric (AUC or AUPRC)

Training separate meta-models for each classifier family is intentionally deferred until later project phases.

---

# Validation Strategy

## Leave-One-Dataset-Out (LODO)

The evaluation uses Leave-One-Dataset-Out Cross Validation.

Example:

```text
Iteration 1

Train:
Datasets 1–16

Test:
Dataset 17
```

Next iteration:

```text
Train:
Datasets 1–15 + Dataset 17

Test:
Dataset 16
```

Repeat until every dataset has served once as the held-out test dataset.

---

## Why LODO?

Random k-fold validation would allow corrupted versions of the same dataset to appear in both training and testing.

This creates information leakage.

LODO guarantees that the meta-model is always evaluated on completely unseen datasets.

---

# Hypothesis Evaluation

LODO verifies that the model generalizes.

However, the research hypothesis is more specific.

The project claims that the proposed method performs particularly well under:

- covariate shift,
- concept shift,
- high corruption severity,
- minimal label distribution change.

Therefore, results are additionally reported by:

```text
Mechanism Class

×

Severity Band
```

This stratified analysis determines whether the method succeeds in exactly the scenarios predicted by the hypothesis.

---

# Research Hypotheses

## H1a

The proposed meta-model predicts Relative ΔM more accurately than a drift-only meta-model.

---

## H1b

The proposed meta-model predicts Relative ΔM more accurately than existing label-free performance estimation methods.

---

## H0

No statistically significant improvement exists over the corresponding comparator.

Support for H1 requires:

- lower MAE, and
- statistically significant improvement (Wilcoxon Signed-Rank Test, p < 0.05).

---

# Baseline Tiers

## Tier 0 — Simple Baselines

Examples:

- historical mean degradation
- single-metric regression (e.g., PSI only)

Purpose:

Provide a minimum acceptable performance level.

---

## Tier 1 — Internal Ablations

Evaluate different feature combinations:

- Quality only
- Drift only
- Quality + Drift
- Quality + Drift + Provenance

Purpose:

Measure the contribution of each feature group.

---

## Tier 2 — External Baselines

Compare against existing methods.

Examples:

- NannyML CBPE
- NannyML DLE
- ATC
- DoC

Purpose:

Measure competitiveness against the current state of the art.

---

## Fair Comparison

Every baseline is evaluated using:

- identical train/test splits,
- identical corrupted datasets,
- identical LODO folds.

No baseline receives a more favorable evaluation protocol.

---

# Statistical Testing

Performance differences are evaluated using the **Wilcoxon Signed-Rank Test**.

Significance threshold:

```text
α = 0.05
```

The comparison is performed on paired per-dataset MAE values generated during LODO evaluation.

This avoids inflated significance caused by treating multiple corruption experiments from the same dataset as independent observations.

---

# Evaluation Metrics

## Primary Metric

### Mean Absolute Error (MAE)

Primary criterion for comparing methods.

Lower MAE indicates more accurate Relative ΔM prediction.

---

## Secondary Regression Metrics

- RMSE
- R²

These provide additional information but do not determine ranking.

---

## Ranking Metric

### Spearman Rank Correlation

Measures how well the predicted degradation ordering matches the true degradation ordering.

---

## Detection Metrics

Applied after thresholding Relative ΔM.

Reported metrics:

- AUROC
- F1 Score

Thresholds:

- 1%
- 3%
- 5%
- 10%

---

# Threats to Validity

## Internal Validity

Prevented by D2's split-then-corrupt protocol, ensuring complete separation between training and testing data.

---

## Construct Validity

Synthetic corruption may not perfectly represent real-world degradation.

This limitation will be investigated in D5 through comparisons with naturally occurring data quality degradation.

---

## External Validity

The benchmark is limited to tabular binary (or near-binary) classification datasets.

Results should not be generalized to other data modalities without additional validation.

---

## Statistical Conclusion Validity

The Wilcoxon Signed-Rank Test is selected because LODO produces only 17 paired observations, making non-parametric testing more appropriate than a t-test.

---

# Deferred Work

Group-aware conformal prediction interval evaluation is outside the scope of D11.

It will be specified in a later Phase 2 design document.

---

# Final Decisions

## Decision 1

Use **Relative ΔM** as the primary prediction target.

---

## Decision 2

Train the meta-model using continuous regression rather than binary classification.

---

## Decision 3

Use Leave-One-Dataset-Out Cross Validation together with mechanism × severity stratified evaluation.

---

## Decision 4

Train one pooled meta-model using all datasets.

---

## Decision 5

Evaluate against three baseline tiers:

- Tier 0 (simple)
- Tier 1 (ablations)
- Tier 2 (state-of-the-art)

---

## Decision 6

Research hypotheses require:

- lower MAE,
- Wilcoxon p < 0.05.

---

## Decision 7

Use the Wilcoxon Signed-Rank Test with α = 0.05.

---

## Decision 8

Compute one deterministic clean baseline (Mclean) for each dataset and model family.

---

## Decision 9

Store every corruption experiment independently.

No averaging across seeds is performed.

---

## Decision 10

MAE is the primary evaluation metric.

RMSE, R², Spearman, AUROC, and F1 are secondary metrics.

---

## Decision 11

Group-aware conformal prediction evaluation is deferred to a future design document.

---

## Decision 12

Future validation documents should adopt Relative ΔM to remain consistent with D11.
