## 1. Overview

D1 defines the corruption engine's contract: given a dataset fold, a mechanism, a severity, and a seed, produce a corrupted version of that fold plus metadata describing what was done. D3 is the implementation of that contract. Six mechanism classes were built, matching D1's six sections:

| D1 Section               | Module                   | Sub-mechanisms                                                   | Status                     |
| ------------------------ | ------------------------ | ---------------------------------------------------------------- | -------------------------- |
| §4 Missingness           | `missingness.py`         | MCAR, MAR, MNAR                                                  | Built, real-data confirmed |
| §5 Feature Noise         | `feature_noise.py`       | Additive Gaussian, Categorical Swap, Quantization                | Built, real-data confirmed |
| §6 Label Noise           | `label_noise.py`         | Uniform, Class-Conditional (Boundary-Aware deferred per D1 §6.3) | Built, real-data confirmed |
| §7 Distribution Shift    | `distribution_shift.py`  | Covariate, Label, Slice-Conditional Concept                      | Built, real-data confirmed |
| §8 Validity Violations   | `validity_violations.py` | Constraint-based (no_negative / range / none)                    | Built, real-data confirmed |
| §9 Uniqueness Violations | `uniqueness.py`          | Global Duplication, Slice-Conditional Duplication                | Built, real-data confirmed |

**14 of 14 sub-mechanisms implemented** (boundary-aware label noise is the one D1-scoped exception, deliberately deferred by D1 §6.3 itself, not an omission).

Every module was built following the same discipline: implement against D1's mathematical specification, test against synthetic data covering edge cases, then test against the real 17-dataset processed corpus. That last step mattered more than expected — four of the six modules had a genuine bug that only surfaced on real data, invisible to any amount of synthetic testing because the bugs depended on properties real data has and hand-constructed test data didn't (pre-existing missingness, rare categories, correlated outlier-heavy features).

D3 went through **two distinct eras**, separated by D4's completion:

- **Era 1 — Initial build and validation (placeholder grid).** All six modules built against D1's specification, tested synthetically, then validated against the real corpus using an illustrative placeholder severity grid (`[0.05, 0.10, 0.15, 0.20, 0.25]`), 3 seeds. Nine real issues found and fixed during this era (§3, §10).
- **Era 2 — Severity-grid integration and re-validation (D4's real grid).** Once D4 finalized actual per-mechanism severity values (10 levels, corpus-wide or per-dataset `α_max` depending on mechanism), `severity_table.csv` was generated and wired into all six runners, replacing the placeholder. This second pass — covering a much wider severity range (up to 0.99 for quantization and label shift specifically) — surfaced two additional serious bugs and one methodological correction invisible to the narrower placeholder range, plus a subtler cross-cutting random-seed correlation bug affecting every module simultaneously (§4, §10).

## 2. Architecture

### 2.1 Interface Contract

Every module exposes the same shape, agreed before any code was written:

```
corrupt(fold, mechanism, severity, seed, train_fold_reference, ...) -> (corrupted_fold, metadata)
```

- `fold` — the fold to corrupt. Per D2 Decision 2, callers are responsible for passing only the **test fold**; no module has an opinion on which fold it's given.
- `train_fold_reference` — the clean train fold, used **only** for parameter estimation (D2 Decision 4: conditioning-feature selection, z-score reference statistics, empirical marginals, bin edges, PC1 direction, IQR bounds, frozen-slice bounds). Never itself corrupted or read from for anything but reference statistics.
- `metadata` — a `CorruptionMetadata` object (defined once, in `base.py`, shared by every module): dataset name, mechanism, severity, seed, achieved rate(s), excluded columns (if any), a validation-passed flag, and a `validation_details` dict with mechanism-specific diagnostics.

### 2.2 Shared Infrastructure (`base.py`)

Started small (just the metadata schema and two math primitives for `missingness.py`) and grew as genuinely shared logic was identified across modules, rather than being duplicated:

- **`CorruptionMetadata`** — the shared return-value schema, including `excluded_columns` (added when per-column exclusion was introduced) and `conditioning_feature` (reused across missingness, distribution shift, and uniqueness for different purposes).
- **`sigmoid`, `reference_zscore`, `solve_logit_offset_for_target_rate`** — math primitives for missingness's MAR/MNAR calibration.
- **`rate_tolerance`, `reference_variance_issue`, `nan_adjusted_target`** — shared rate/variance/NaN logic, moved here from `missingness.py` once `feature_noise.py` needed the identical logic (avoiding two copies drifting apart, the same single-source-of-truth concern raised for the runner scripts' `TARGET_COLUMNS` mapping).
- **`select_frozen_slice_bounds` / `select_frozen_slice`** — the shared, deterministic frozen-slice function, built specifically so `distribution_shift.py` (§7.3) and `uniqueness.py` (§9.2) could provably operate on the identical slice S, per D1's requirement that the two mechanisms share one frozen region. Deterministic-and-cached rather than persisted-to-disk, deliberately: a stale on-disk slice surviving a code change would silently violate "frozen from this fold," which is exactly the kind of undocumented cross-run state this project's audit philosophy exists to prevent.
- **`derive_call_seed(seed, dataset_name, mechanism, severity)`** — added in Era 2 (§4.3). Derives a unique per-call seed from the full call context via SHA-256 hashing, replacing every module's direct use of the caller's raw seed index with `np.random.default_rng()`. This is the fix for the seed-correlation bug described in §4.3.

## 3. Era 1 — Module-by-Module Build and Validation (Placeholder Grid)

Full narrative: what was built, how it was tested, and exactly what went wrong and why. Era 2's re-validation findings under D4's real grid are in §4; the consolidated bug log across both eras is in §10.

### 3.1 Module 1 — `missingness.py` (D1 §4): MCAR, MAR, MNAR

**Mechanisms:** MCAR (`P(R=0) = π_j(α)`), MAR (`P(R=0) = σ(α·z(X_ik))`, conditioning feature k selected by highest train-fold correlation), MNAR (`P(R=0) = σ(α·z(X_ij))`, depends on the column's own value).

**Design notes.**

- D1's MAR/MNAR formula has no intercept term. Taken literally, `σ(0) = 0.5` at `z=0` regardless of severity — meaning even near-zero severity would produce ~50% missingness for typical rows. A calibration offset (solved via bisection so the achieved rate matches the target) was added to fill this gap, flagged explicitly as a D3 decision rather than a reinterpretation of D1's math.
- MNAR's mechanism-fidelity check (missingness significantly associated with the feature's own pre-corruption value) is explicitly **blocking** per D1 §4.3 — a draw that reduces to MCAR-equivalent behavior is invalid, not merely suboptimal.

**Testing journey and bugs found.**

- **Round 1 — tolerance miscalibration.** The first real-corpus run (`--datasets phoneme churn`) showed 10/90 rows with `validation_passed=False`, concentrated at higher severities. Initial hypothesis (a real RNG bias at a specific call position) was tested against 500 seeds and disproven — pure coincidence from only testing 3 seeds. The actual issue: a flat `RATE_TOLERANCE = 0.02` tolerance doesn't scale with the true sampling noise, which follows `sqrt(severity·(1-severity)/n)`. At low severity the flat band was too loose; at high severity, too tight — producing a severity-dependent false-positive rate (~11% observed). Fixed with a binomial-standard-error-based tolerance (`RATE_TOLERANCE_Z = 3.5`), re-verified at ~1% false-positive rate.
- **Round 2 — pre-existing NaN propagation.** Running against the full corpus, Airbnb's MAR and MNAR results showed achieved rates of exactly `0.0` for most columns at every severity — not noise, a complete failure to inject any missingness. Root cause: Airbnb has substantial genuine pre-existing missingness (a documented Phase 0 realism anchor), and when the conditioning/target column already has NaN, `reference_zscore` produces NaN in the z-array. The calibration bisection's `sigmoid(...).mean()` then returns NaN (numpy's NaN propagation), and every comparison against NaN evaluates False — so the bisection always took the same branch and collapsed to its -50 floor regardless of target severity. Fixed by calibrating against only non-NaN z-values, plus a `nan_adjusted_target` helper that scales the calibration target so the achieved rate (measured across all rows, including permanently-ineligible NaN ones) still lands on the requested severity. A related fix was needed in the association test (`scipy.stats.pointbiserialr` doesn't handle NaN internally).
- **Round 3 — EEG's all-or-nothing blocking design.** Running `--all`, EEG's MNAR showed 0/15 — every draw failed. Initial hypothesis (a near-constant reference column) was tested and disproven directly against the real correlation data: AF3 showed a real, weak, borderline-significant correlation (0.005 to 0.04, occasionally crossing significance), not zero. The actual issue was structural: the blocking check required every corrupted column in a call to pass, so one persistently borderline column (AF3) invalidated four other columns (F7, F3, FC5, T7) that were working correctly every time. Redesigned to per-column granularity: the fidelity check now runs before the mask is applied, per column; a failing column is excluded from that call's output (and reported in `excluded_columns`) rather than invalidating the whole call. Raises only if every requested column fails.

> **Amendment, 2026-07-22 (dated correction, original text above left unchanged):** the framing above characterizes AF3 as the one uniquely borderline column, with F7/F3/FC5/T7 as incidental casualties of the all-or-nothing blocking design. D6's independent per-channel skew/std profiling pass (D6 §2.4.2) found this framing incomplete: **seven** channels — `FC5, O1, P7, P8, AF3, F8, AF4` — share a nearly identical, extreme skew signature (118–122), too tight a cluster to be coincidental, consistent with a shared artifact event (e.g. simultaneous electrode saturation or a movement artifact) affecting several electrodes at once, rather than AF3 behaving badly in isolation. **Corrected framing: "EEG has a coherent multi-channel artifact signature affecting 7 of 14 channels," not "AF3 is uniquely problematic."** This does not change any of D3's engineering conclusions — the per-column exclusion redesign remains the correct fix regardless of how many channels are affected — but the _diagnostic narrative_ (why AF3 in particular looked borderline) should be read alongside D6 §2.4.2 and §11 below.

**Era 1 result:** 765/765 calls completed (750 OK, 15 PARTIAL_SUCCESS — all in EEG, expected given its documented borderline columns), 0 unexpected errors.

### 3.2 Module 2 — `feature_noise.py` (D1 §5): Gaussian, Categorical Swap, Quantization

**Mechanisms:** Additive Gaussian Noise (`X' = X + ε`, `ε ~ N(0, (α·s_j)²)`, applied to continuous features only), Categorical Swap (resample from the empirical marginal excluding the cell's own value), Quantization / Bin Collapse (`k = k_max − α·(k_max − k_min)`, equal-frequency binning).

**Design notes.**

- D1 restricts Gaussian noise to continuous features but doesn't specify `k_max` for quantization. Set per-column as `min(train-fold distinct value count, 20)` — flagged as a D3 decision, `QUANTIZATION_K_MAX_CAP = 20`.
- Quantization is structurally different from every other mechanism in the codebase: severity controls coarseness applied uniformly to every value, not a per-cell selection rate. No `achieved_rate`-as-proportion; the metadata instead reports achieved vs. target bin count. `seed` is accepted for interface consistency but genuinely unused (the mechanism is deterministic given the reference and k).
- Pre-existing NaN required no special handling for Gaussian noise — `NaN + noise = NaN` falls out of ordinary floating-point arithmetic.

**Testing journey and bugs found — chi-square bin-dropping crash.** The first real-corpus `--all` run showed 150/765 draws crashing with a scipy error: "the sum of the observed frequencies must agree with the sum of the expected frequencies." The pattern was deterministic, not intermittent — every dataset with at least one low-frequency category in its categorical marginal failed 15/15; every dataset without one (electricity, bank-marketing, ACSPublicCoverage) passed 15/15. Root cause: the marginal-preservation chi-square check dropped bins with expected count under 5 via boolean indexing before calling `scipy.stats.chisquare` — but dropping bins breaks scipy's required invariant that observed and expected totals match exactly. Fixed by pooling low-expected-count categories into a single combined "other" bin instead of dropping them, which preserves both totals exactly while still satisfying the standard expected-count-of-5-or-more rule of thumb. Re-verified: reproduced the exact crash with a forced rare category (expected count ~2), confirmed the pool triggers and zero crashes result; re-ran the full corpus and all 150 previously-crashing draws passed.

**Era 1 result:** Gaussian and Quantization clean on their very first real run (255/255 each). Categorical Swap: initially 45 OK / 150 crashed / 4 skipped (no eligible columns); after the fix, 705 OK plus 4 skipped, 0 crashes, 0 partial successes.

### 3.3 Module 3 — `label_noise.py` (D1 §6): Uniform, Class-Conditional

**Mechanisms:** Uniform (`P(Y'≠Y) = α`, symmetric), Class-Conditional (asymmetric directional rates, ratio anchored to a Cleanlab confident-learning pass per D1 §6.2 — never asserted from intuition). Boundary-Aware (D1 §6.3) is explicitly deferred, not implemented — calling it raises `NotImplementedError` with the D1 rationale (probe/evaluation-family circularity risk) rather than silently doing nothing.

**Design notes.**

- Severity is capped below 0.5 for binary labels — at or above a 50% flip rate, Y becomes independent of X, which is total signal destruction, not "high severity." This is **enforced**, not just documented.
- If Cleanlab is unavailable, class-conditional noise raises an ImportError rather than silently falling back to a symmetric ratio — a symmetric fallback would quietly reduce class-conditional noise to uniform noise, defeating the mechanism and violating D1 §6.2's anchoring requirement.
- A known, flagged tension: the Cleanlab estimate uses logistic regression as its label-quality probe, and logistic regression is one of the four evaluated model families. The circularity risk is much milder here than D1 §6.3's boundary-placement concern (this only estimates a single scalar ratio), but it's real and worth revisiting when the final pipeline is assembled. (See also §4.2 of D4's build report, where this same probe-underfitting property invalidated the Cleanlab-estimated _overall rate_ — a distinct parameter from the asymmetry ratio.)

**Testing journey and bugs found.**

- **Wrong Cleanlab API name.** The first attempt to test the real Cleanlab estimation path failed with an ImportError even though Cleanlab was installed. The guard import referenced a function that doesn't exist in the installed Cleanlab version (2.9) — the correct function is `compute_confident_joint`. Fixed by correcting the import; re-verified the estimation path works end-to-end, including a sanity check that the estimated ratio responds correctly to injected asymmetric label errors.
- **Roughly fifteen-times redundant computation.** After the real-corpus run showed 510/510 OK, a closer look revealed the Cleanlab estimate — an expensive 3-fold cross-validated logistic regression fit — was being recomputed from scratch on every one of 15 severity/seed calls per dataset, even though the estimate depends only on `train_fold_reference`, `target_col`, and `feature_cols`, none of which vary across severity or seed. Fixed with a cache keyed on **object identity** of `train_fold_reference` plus `target_col` and `feature_cols` — object identity rather than dataset name, since dataset name is caller-supplied and could be blank or accidentally reused, whereas object identity is correct as long as the same DataFrame object is reused across calls (exactly the runner's actual usage pattern). Verified directly: 15 calls with the same train fold now produce exactly 1 "fresh" computation and 14 "cached" hits, ratio provably identical across all 15; a genuinely different DataFrame object correctly triggers a fresh computation. Measured speedup: ~15× at smoke-test scale (0.94s vs. ~14.15s per dataset); confirmed against the real corpus manifest afterward — every one of the 17 datasets showed exactly 1 fresh + 14 cached computations, exactly as predicted.

**Era 1 result:** 510/510 OK (17 × 2 mechanisms × 5 severities × 3 seeds). 509/510 `validation_passed=True`; the one miss (default_credit, class-conditional, severity=0.1, seed=3) traced to a 3.51σ deviation against a 3.5σ tolerance — a boundary-case statistical miss, not a defect. One dataset (`online_shoppers`) showed a notably higher estimated asymmetry ratio (6.73 vs. 0.6–3.7 for the rest) — flagged as worth a second look, not yet resolved (see §13).

### 3.4 Module 4 — `distribution_shift.py` (D1 §7): Covariate, Label, Concept

**Mechanisms:** Covariate Shift (PC1-weighted importance resampling, `P(select) ∝ exp(α·g(x))`), Label Shift (class-stratified resampling toward a target prior), Slice-Conditional Concept Shift (label flips restricted to the frozen slice S, features untouched).

**Design notes.**

- Covariate shift is implemented as weighted resampling with replacement of the fold's own rows, rather than any transformation of feature values. This is what makes D1's preservation requirements (`P(Y|X)` for covariate shift, `P(X|Y)` for label shift) hold **by construction**: each resampled row keeps its own original `(x, y)` pairing (covariate shift) or is drawn only from within its own class (label shift), so the conditional relationship literally cannot change.
- PC1 is fit on the train fold and deliberately left as the sole generator — the direction of maximum variance, not the direction most predictive of Y. This is intentional per D1 §7.1: a benchmark where drift magnitude and degradation magnitude always moved together would be trivially solvable.
- Concept shift is the one sub-mechanism confirmed to use the shared frozen slice S from `base.py`, not an independently computed one — see §7 for the verification.

**Testing journey and bugs found — degenerate resample collapse (the most serious bug found across all six modules).** The real-corpus run showed every dataset's covariate-shift PSI rising smoothly and monotonically with severity, except jm1: 0.045, 2.300, 7.326, 7.895, 7.895 — an order of magnitude beyond every other dataset, and the exact repeated value at the two highest severities was the tell that something had saturated rather than merely drifted. Diagnosed directly, not guessed at, by instrumenting the actual weight computation: jm1's software-complexity features (`loc`, `v(g)`, `ev(g)`, `iv(g)`, `n`) are correlated and outlier-heavy, and the exponential weighting formula is mathematically unbounded — when PC1 is dominated by even one or two extreme points, the exponential can put effectively 100% of the resampling probability mass on a single row. Confirmed precisely: effective sample size dropped to 1.0 out of 2000 by severity 0.15. The "corrupted" fold at that point was not a shifted distribution, it was ~2000 duplicates of one row — which would have silently produced meaningless downstream results for jm1's covariate-shift condition while every status check reported complete success. Fixed by clipping the PC1 score to a bounded number of its own standard deviations (`G_CLIP_Z = 6.0`) before exponentiating, plus an effective-sample-size diagnostic added to metadata. Re-verified: jm1's PSI curve became 0.020, 0.070, 0.246, 0.565, 1.186 — smooth and in the same range as heloc and default_credit. **This Era-1 fix left a milder residual (jm1 still tripping the ESS floor at severity 0.25, fraction 0.046, reproducible across all 3 seeds) and was later found incomplete — fully resolved in Era 2 §4.2.** A related runner-script gap was found and fixed in the same pass: the ESS diagnostic fields were computed but never written to the manifest CSV; fixed to write them explicitly, plus live console logging when a resample is flagged degenerate.

**Era 1 result:** 765/765 OK. All 17 datasets show smooth, monotonically increasing PSI with severity, with jm1's severity-0.25 residual the only non-trivial validation outcome (subsequently eliminated in Era 2).

### 3.5 Module 5 — `validity_violations.py` (D1 §8): Constraint-based

**Mechanism:** A cell's value is replaced with an invalid value for a proportion of selected cells, where the invalid value is chosen per-column from a one-time, human-confirmed domain-constraint annotation (`no_negative`: negate; `range` (bounded interval): push beyond whichever bound; `none` (no documented constraint): push beyond a train-fold IQR-based bound; `categorical`: excluded entirely, not a numeric magnitude constraint).

**Design notes.**

- This module's build was blocked for several turns pending D1 Decision Log item 6 (a one-time domain-constraint annotation pass, owned by Kishore) and a separate naming inconsistency (`road-safety` vs. `road_safety` duplicate entries in an earlier draft of the annotation file). Both were confirmed resolved directly by inspecting the actual uploaded annotation file: all 17 datasets present, zero blank confirmations across 369 rows, and only the canonical `road-safety` name present. **175 of 369 rows (47%)** showed the confirmed constraint type differing from the auto-suggested one — strong evidence the annotation pass was a genuine manual review, not a rubber stamp. Notably, EEG's channels were corrected from the auto-guessed `no_negative` to `none`, with the explicit note that observed non-negativity was a sampling artifact, not a real device constraint — directly consistent with what real-data testing of `missingness.py` had already suggested about EEG's channel behavior.
- The `no_negative` violation includes a guard: if negating a value would still not produce something below zero (e.g. a pre-existing negative value in an otherwise-positive column), the value is pushed further negative rather than silently producing a non-violation.
- Two new D3 constants, both flagged as undocumented by D1: `IQR_MULTIPLIER = 3.0` (deliberately more extreme than the standard 1.5× mild-outlier convention, since the injected value needs to read as implausible, not just statistically unusual) and `RANGE_OVERSHOOT_FRACTION = 0.10`.

**Testing journey.** This is the one module that passed its first real-corpus run cleanly — no bugs found. Verified directly against all 17 datasets' actual constraint sets from the real annotation file, not just a synthetic subset: correct parsing of every real constraint format, correct exclusion of categorical columns, and genuinely constraint-violating injected values (e.g. latitude pushed to exactly ±108, outside the range −90 to 90 by the configured overshoot).

**Era 1 result:** 255/255 OK. 253/255 `validation_passed=True`; the 2 misses (Airbnb, MagicTelescope) traced to 3.5–3.8σ deviations, and notably landed on exactly the two datasets with the most columns checked per call (37 and 10 respectively), consistent with a multiple-comparisons effect rather than any defect.

### 3.6 Module 6 — `uniqueness.py` (D1 §9): Global, Slice-Conditional Duplication

**Mechanisms:** Global Duplication (a computed count of rows sampled with replacement, appended as exact duplicates, no jitter), Slice-Conditional Duplication (same mechanics, restricted to the frozen slice S).

**Design notes.**

- Structurally unique among all six modules: the corrupted output is **larger** than the input, since rows are appended, not transformed in place. Every other mechanism preserves row count. (D7 implication in §12.)
- `validation_passed` here is deterministic, not statistical — the appended count is computed directly, not achieved via independent per-row Bernoulli draws the way rate-based mechanisms elsewhere are. A clean run means the row math held exactly, not "close enough within tolerance."
- Baseline vs. injected duplicate rate are deliberately kept separate in the metadata: several corpus datasets (jm1, heloc) have documented natural duplicates from Phase 0, and generic post-hoc duplicate detection on the corrupted output would conflate pre-existing duplicates with newly-injected ones, overstating what the mechanism actually did.
- The slice-conditional variant obtains the frozen slice S from `base.py`'s shared function — the identical function `distribution_shift.py`'s concept shift uses — rather than computing its own.

**Testing and cross-mechanism verification.** No bugs found — the module's core logic (direct count computation, exact row appending) left little room for the subtle statistical or NaN-propagation bugs found elsewhere. The one property that mattered most to verify wasn't correctness of this module in isolation, but its alignment with `distribution_shift.py`: both modules were called independently with identical inputs and confirmed to return byte-identical slice bounds, down to the exact floating-point value (see §7).

**Era 1 result:** 510/510 OK. Real natural-duplicate-rate measurements captured for all 17 datasets as a side effect of the runner's diagnostic logging (see §6.3).

## 4. Era 2 — Severity-Grid Integration and Re-Validation (D4's Real Grid)

Switching from the placeholder grid (max severity 0.25) to D4's real grid (max severity up to 0.99 for several mechanisms) exercised code paths the placeholder range never reached, surfacing two further serious bugs and one cross-cutting bug. By the end of Era 2, all six modules are confirmed clean under both the final seeding scheme and D4's final severity grid.

### 4.1 `feature_noise.py` — Quantization's bin-count tolerance was too strict for real data

**Symptom.** 285 of 1414 real-corpus draws (20%) failed validation, 255 of them in `quantization` alone — roughly 50% of all quantization draws, spread evenly across every severity level (not concentrated at high severity, ruling out an initial hypothesis about zero-inflated columns alone).

**Diagnosis.** The original tolerance required `achieved_k` within `±1` of `target_k`. Reproduced directly: a column like ACSIncome's `SCHL` (education level — an ordinal variable with ~24 distinct codes but heavily uneven frequencies) earns a high `k_max` from its raw distinct-value count, but `pd.qcut`'s equal-frequency binning can only _merge_ requested bins when values are lumpy — it can never produce more bins than requested, only fewer. For `SCHL`, `target_k=18` but `achieved_k=7` — a legitimate, benign consequence of the data's actual shape, not a defect. The fixed `±1` tolerance had no way to distinguish this from a genuine failure.

**Fix.** Replaced the fixed tolerance with a structural check: `achieved_k <= target_k` (always true by construction) `and achieved_k >= QUANTIZATION_K_MIN` (catches genuine degenerate collapse — total information loss for a column — while accepting legitimate, data-driven undershoot).

**Re-verified against real data.** Quantization failures dropped from 255 to 84, and — checked directly — **every single one of the 84 remaining failures has at least one column with `achieved_k=1`**, a genuine degenerate collapse, correctly caught by the new floor. Zero false positives remain.

**Related open finding (not fixed).** `diabetes_readmission`'s `categorical_swap` on the `weight` column shows an achieved rate frozen at the exact same value (`0.031064595693620425`, full float precision) across every severity from 0.06 to 0.30. Root cause: `weight` is 96.82% naturally missing (from D4's profiling), so `nan_adjusted_target`'s severity-scaling saturates near 100% of the ~3.1% eligible cells almost immediately — requesting any severity above that ceiling is asking for something physically impossible, and it's currently caught only as a post-hoc validation failure rather than a pre-emptive exclusion (the same category of gap as the quantization case, in a different mechanism). Documented as open (see §13).

### 4.2 `distribution_shift.py` — The original jm1 fix (`G_CLIP_Z`) was incomplete; EEG exposed a deeper flaw

**Symptom.** EEG's covariate shift showed `effective_sample_fraction = 0.000` at **every single severity level tested, immediately**, with PSI jumping to 5.365 at the very lowest severity (0.025) and plateauing at 8.185 — worse than jm1's original bug (which degraded gradually and left a residual 4.6% floor even after the Era 1 fix) and structurally different (instant and total, not gradual).

**First diagnosis attempt (wrong, corrected before delivering — kept in the record deliberately).** Hypothesized this was the same "outlier-dominated PC1" issue as jm1, requiring the same fix (exclude near-constant reference columns). Implemented an exclusion using the existing `reference_variance_issue` check (`MIN_REFERENCE_STD = 1e-8`). **Tested it — it didn't work.** The likely-responsible EEG channel (probably `AF3`) has a train-fold standard deviation around `1e-4` — non-degenerate by the `1e-8` threshold, so it wasn't excluded, and the collapse persisted identically.

**Correct diagnosis.** The real problem isn't that the column is _constant_ — it's that its scale is _disproportionately tiny relative to the other features_. When z-scored, a handful of genuine small real-world artifacts (entirely plausible in raw EEG signal) produce enormous standardized values that dominate PC1's loading. Critically, `G_CLIP_Z`'s clip bound is computed as a multiple of `g_centered`'s own standard deviation — but that same standard deviation is inflated by the very outliers it's supposed to bound against. A non-robust statistic chasing its own outliers: clipping to `6 × std` doesn't help when a few extreme points have already dragged `std` up to match them.

**Correct fix.** Replaced the non-robust standard deviation with a MAD-based robust scale estimate (`1.4826 × median(|x − median(x)|)`) for the clip bound. Reproduced the exact EEG failure synthetically, confirmed the new clipping resolves it completely (effective sample fraction 99.9% → 93.4% across the full severity range, smooth and graceful, PSI 0.008 → 0.073), re-ran the full regression suite (well-behaved data still monotonic; **jm1's original case now _fully_ healthy — 100% effective sample size at severity 0.25, an improvement over Era 1's residual 4.6% floor, not just a restoration**).

**Re-verified against real data.** EEG's real-corpus PSI curve is now `0.001 → 0.019`, smooth and monotonic — indistinguishable from any other well-behaved dataset, with zero degenerate-resample flags anywhere in the run.

**Secondary question, investigated and resolved as a non-issue.** Three datasets (`churn`, `jm1`, `phoneme`) showed occasional PSI non-monotonicity warnings in the same runs. Checked directly across all three seeds for each: the dip location and presence varied unpredictably by seed (e.g. `churn` non-monotone only at seed 1, monotone at seeds 2 and 3; `phoneme` non-monotone at seeds 1 and 3, monotone at seed 2) — the signature of ordinary PSI-estimation sampling noise on the corpus's smaller test folds, not a systematic mechanism flaw. No fix needed.

### 4.3 The random-seed correlation bug (cross-cutting: all six modules plus the runner scripts)

**Symptom.** After `missingness.py`'s Era-1 fixes were confirmed clean, a closer statistical look at a routine-looking real-corpus run (1467 OK, only 12 `validation_passed=False` out of 1530) found something that shouldn't have been there: **all 12 failures landed on seed 3**, none on seeds 1 or 2. Expected false-positive count under pure statistical noise: ~3.6. Probability of all 12 landing on one specific seed by chance, if seed were irrelevant: **5.65×10⁻⁶**.

**Diagnosis.** Every module created `np.random.default_rng(seed)` fresh, per call, using the caller's raw seed index (1, 2, or 3) directly. Because the seed pool is small and reused across every dataset/mechanism/severity combination, every call sharing "seed=3" draws from the _same starting prefix_ of that one fixed PRNG sequence, just truncated to different array lengths (`n`). These are not fully independent draws, even though the experimental design treats them as if they were. Confirmed directly: simulating the actual battery of `(n, severity)` combinations used in the real runs, seed 3 showed 72.2% positive deviations vs. seed 1's 27.8% — a stark, mechanistically-explained bias, not noise.

**Fix.** `derive_call_seed(seed, dataset_name, mechanism, severity)` — hashes the full call context into a new seed via SHA-256, eliminating the shared-prefix correlation while remaining fully deterministic and reproducible. Applied to every `np.random.default_rng(seed)` call site across all six corruption-engine modules (11 sites total). Re-verified: seed 3's bias dropped from 72.2% to 41.1% positive deviations — no longer distinguishable from seeds 1 (51.1%) or 2 (55.6%).

**Confirmed on real data (missingness, before vs. after):**

|                                 | Before         | After                 |
| ------------------------------- | -------------- | --------------------- |
| `validation_passed=False` count | 12             | 1                     |
| Seed concentration              | 100% on seed 3 | None (1 miss, seed 2) |

**A related, structurally identical instance found independently, not by re-running the same statistical test.** The six runner scripts' `split_train_test` function used the same anti-pattern: `SPLIT_SEED = 42`, one fixed literal value, reused identically across all 17 datasets, with a fresh generator created from it each time. Flagged by inspection rather than by a demonstrated statistical anomaly — applied the same `derive_call_seed` fix as a justified precaution, though without the same direct "before vs. after" proof the corruption-engine fix had. Fixed in all six runners; re-verified functionally (all six run cleanly, reproducibility holds), though not via the same seed-bias statistical test, since there's no natural way to compare a single fixed literal seed against itself.

**Scope of this fix:** `base.py` (new `derive_call_seed` function), all five remaining corruption-engine modules (`feature_noise.py`, `label_noise.py`, `distribution_shift.py`, `validity_violations.py`, `uniqueness.py` — 10 additional call sites), and all six runner scripts' `split_train_test` functions.

## 6. Real Dataset-Specific Findings

These came directly out of validating D3 against the actual corpus, not out of any design document. Worth carrying into D4's per-dataset severity calibration rather than treating every dataset identically.

### 6.1 EEG's channel structure (missingness, MAR/MNAR)

Of EEG's 5 auto-selected channels, `F7`, `F3`, `FC5`, `T7` show strong, consistently significant self/cross-dependency; `AF3` shows a real but weak, often-borderline self-dependency that intermittently fails MNAR's significance check (correctly excluded per-column, not blocking the other four). For MAR specifically, cross-channel correlations among most EEG channel pairs are close to zero — only `F7` reliably passes as a conditioning-feature target. Consistent with EEG channels being largely independent raw signals at the amplitude level. No action needed for missingness; documented for awareness if EEG's missingness behavior looks asymmetric across mechanisms in later results. **See the 2026-07-22 amendment in §3.1 and §11 for the reconciliation of AF3's "uniquely borderline" framing against D6's 7-channel skew finding.**

### 6.2 jm1's covariate-shift behavior (distribution_shift, §7.1)

jm1's software-complexity features (`loc`, `v(g)`, `ev(g)`, `iv(g)`, `n`) are correlated and outlier-heavy (a few large/complex files among mostly small ones). Before the `G_CLIP_Z` fix, this caused PC1-weighted resampling to collapse to duplicates of a single row by severity 0.15 (PSI=7.9, effective sample size ~0.05%). After the Era-1 fix, jm1 behaved like the rest of the corpus at low-to-moderate severity but still tripped the effective-sample-size floor at severity=0.25 specifically (effective_sample_fraction=0.046, all 3 seeds, fully reproducible; PSI at that point a reasonable 1.19, not degenerate — real diversity reduced but not collapsed). **This residual was fully eliminated in Era 2 by the MAD-based robust clipping fix (§4.2): jm1 now reaches 100% effective sample size at severity 0.25.** The D4 action originally flagged here (cap jm1's covariate-shift severity below 0.25, or accept 4.6% as a disclosed property) was resolved by D4 as "keep the uniform grid, flag in the manifest"; with the Era-2 fix the flag no longer triggers, but the decision stands harmlessly.

## 7. Cross-Mechanism Guarantees

Three structural guarantees, built into the shared infrastructure and explicitly verified rather than asserted:

1. **Frozen slice S sharing (D1 §7.3 ↔ §9.2).** The shared slice function in `base.py` is the single source of truth. Verified: two independent callers (simulating `distribution_shift.py`'s concept shift and `uniqueness.py`'s slice-conditional duplication), given identical `(train_fold_reference, seed, dataset_name)`, receive **byte-identical** slice bounds down to the exact floating-point value. Different seeds produce different slices (confirmed); repeated calls with the same inputs are fully deterministic (confirmed across 5 repetitions). This is what lets D11's stratified evaluation ask whether a given region degrades under concept shift and duplication, or only one. Noted so it isn't accidentally "fixed" into two independent implementations later.
2. **Shared rate/variance/NaN-handling logic (`missingness.py` ↔ `feature_noise.py`).** The rate-tolerance, reference-variance, and NaN-adjustment helpers were moved to `base.py` once both modules needed identical logic, rather than maintaining two copies that could silently drift apart.
3. **Shared seed-derivation logic (new in Era 2).** `derive_call_seed` is used identically across all six modules and all six runners — the same single-source-of-truth principle extended to random-seed handling project-wide.

## 9. Testing Methodology

Every module went through the same sequence, without exception:

1. **Synthetic construction** covering the mechanism's core behavior and known edge cases (pre-existing NaN, constant columns, low-cardinality categories, class-emptying severities).
2. **Direct verification of mathematical properties**, not just whether it ran — e.g. confirming PSI actually rises monotonically with severity, confirming class-conditional noise's directional rates respect the target asymmetry ratio, confirming negated values are genuinely negative.
3. **Real-corpus validation** via the runner scripts, against all 17 actual processed datasets.
4. **Root-cause diagnosis before any fix** — every bug reproduced synthetically first, with the specific data property responsible isolated and confirmed, before writing a fix. Three false starts are part of this record deliberately, not hidden: (a) an initial claim of a real, reproducible RNG bias in missingness, tested against 500 seeds and retracted; (b) an initial near-constant-column diagnosis for EEG's MNAR failure, directly disproven by the real correlation data before the actual cause (all-or-nothing blocking design) was found; (c) the first attempted fix for EEG's covariate-shift collapse (excluding near-constant columns via the `1e-8` threshold), implemented, tested, found not to work, and replaced with the correct MAD-based robust clipping (§4.2) — the same "test before trusting a diagnosis" discipline applied to a fix, not just to bug detection.
5. **Re-verification against real data after every fix**, not just synthetic re-testing — confirming the specific real dataset that surfaced the bug now behaves correctly, in addition to confirming no regression in previously-passing cases.

**Era 2 addition — re-validate at the real severity range.** Era 2 specifically tested under both the placeholder grid and D4's real grid, and the real grid's wider severity range (up to 0.99 vs. 0.25) was what surfaced the quantization and EEG bugs — neither would have been visible under the placeholder range alone. This is now a standing lesson for any future severity-grid change: re-validation at the actual severity range used in Phase 2 is not optional, since bugs can hide comfortably within a narrower testing range.

3. **D5's target definition needs updating for consistency.** Carried forward from D11: D5 (Natural-Shift Validation) was originally scoped around "ΔAUC" curves; should use Relative ΔM instead, per D11 Decision 1/12. _(Resolved: D5 v1.1 now uses Relative ΔM throughout — see D5 §1.)_
